Usage

A scheduler can return a time object with a startDateTime, a timeZone, and arecurrencePattern

Via data-attributes

The scheduler will automatically instantiate any schedulers with data-initialize="scheduler". If you add the control markup after page load with data-initialize="scheduler", this control initializes on mousedown.

Via JavaScript

Call the scheduler control via JavaScript:

{% highlight js %}$('#Myscheduler').scheduler();{% endhighlight %}

Markup

Methods

$().scheduler()

Attaches scheduler to a list of steps, navigation, and content

.scheduler('destroy')

Removes all functionality, jQuery data, and the markup from the DOM. Returns string of control markup.

{% highlight js %}$('#element').scheduler('destroy');{% endhighlight %}

.scheduler('disable')

Disable all scheduler form fields

{% highlight js %}$('#element').scheduler('disable');{% endhighlight %}

.scheduler('enable')

Enables all scheduler form fields

{% highlight js %}$('#element').scheduler('enable');{% endhighlight %}

.scheduler('value')

Gets the current scheduler form field information or sets it, depending on arguments present

{% highlight js %} $('myScheduler').scheduler('value'); {% endhighlight %}
Setter Method

Use 'value' while passing an object following the structure listed below as the second parameter

{% highlight js %} $('myScheduler').scheduler('value', { startDateTime: '2014-03-31T03:23+02:00', timeZone: { name: 'Namibia Standard Time', offset: '+02:00' }, recurrencePattern: 'FREQ=MONTHLY;INTERVAL=6;BYDAY=WE;BYSETPOS=3;UNTIL=20140919;' }); {% endhighlight %}
Parameter Description
startDateTime Required. String representing the start date & time in ISO 0861 format.
timeZone Object containing the time zone name and offset.
recurrencePattern Optional. String representing the iCal recurrence value.
