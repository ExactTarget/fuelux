Repeater Views

The repeater view object contains various attributes used to render data within the repeater control. Each view includes a unique name that is selectable via the .repeater-views classed button group. The $.fn.repeater.views object adds the view object upon load using the unique view name as a key. Find more information on creating a repeater view extension below. You can also review this example.

Default Values and Methods

You can add default values and methods in the same way as general extensions, described above.

Functions

Call all functions within the repeater view object with the repeater instance context of this. The functions receive a helpers object and callback function. The callback function must run to proceed to the next step. You can pass the callback function an object as an argument to perform various tasks specific to each function.

View Object

The main view object serves as the base for the repeater view extension and supports the following attributes:

Name type description
initialize function Runs only once at the begeinning of each instantiation of the repeater. Provided a helpers object and callback function.
selected function Runs every time this repeater view is selected prior to rendering. Provided a helpers object and callback function.
renderer object A renderer object, with various attributes specifying how to render content or data. Provided a helpers object and callback function.

Renderer Object

The renderer object serves to render content or data at a specific level. This object uses the following structure:

Name type description
after function Runs after the code calls the render function and appends the content to the DOM. Provided a helpers object and callback function. The callback accepts an object with the skipNested attribute. If the skipNested value is true, the repeater will ignore rendering the nested array for this renderer.
before function Runs before the render function. Provided a helpers object and callback function. The callback accepts an object with the skipNested attribute. If the skipNested value is true, the repeater will ignore rendering the nested array for this renderer.
complete function Runs after all the renderers in the nested array have ran. Provided a helpers object and callback function.
nested array Array of renderer objects. These objects use the same structure as the one currently being described. Each object will run in order following the render function of the current object unless otherwise specified.
render function Runs after the before function. This function renders data and adds content to the parent container. Provided a helpers object and callback function. The callback function accepts an object with several attributes:
  • item
  • action
  • skipNested
The item attribute represents the desired content to be injected into the DOM as either a string or jQuery object. Item content can include special data-attributes for additional rendering behaviors. The action attribute (string) specifies how the item will inject in relation to the parent container. This value defaults to append, but it can use any valid jQuery DOM injection method, such as 'prepend' or 'after'). The skipNested attribute, if set to true, will tell the repeater to ignore the nested array for the current renderer.
repeat string If set, this renderer will render multiple times over a subset of data. The string value specifies which subset this renderer iterates over. Represents an attribute of an object using dot notation. The root level value must use 'data' or 'this'. If the root value uses 'data', the root object will incldue the data object returned from the dataSource. If the root value uses 'this', the root object will include the current repeater context of this. You can specify further levels separated by dots. The final level must inlcude an array or iteration will not function correctly. This array will pass as the subset attribute in the helpers object for other renderer functions. (ex: 'data.animals.pigeons')

Helpers Object

The helpers object contains various attributes useful for rendering data:

Name type description
container jQuery object The parent container for the current renderer. Default value includes the .repeater-canvas div. Remains true unless another container is specified via data-attributes.
data object The data object as returned from the dataSource in its entirety
index number Only present if the repeat attribute value equals the current renderer. Specifies the current index of the subset being iterated over.
item str or jQuery object Represents the item returned from the render function of the current renderer. Only present in functions following the render function when an item was returned.
subset array Only present if the repeat attribute value equals the current renderer. Specifies the subset iterated over.

Data Attributes

You can add a few data-attributes to any markup when rendering content to specify different rendering behaviors:

Name Value(s) description
data-container "true" If you add this value to an element within a returned item, the element will become the parent container for any nested renderer within that renderer object.
data-infinite "true" If infinite scrolling is enabled, the element with this attribute becomes the infinite scrolling container.
data-preserve "deep" or "shallow" The repeater will preserve an element containing this attribute upon clearing the repeater of data. This clear occurs during each dataSource call. Set the value to "deep" to preserve the element and all children. Set the value to "shallow" to preserve just the element and remove any child nodes not marked with data-preserve.
